# Miri Look Product Roadmap

> 기준일: 2026-06-24  
> 목적: 고객 파일럿 MVP 이후 웹서비스, 관리자, 커뮤니티, 입점, 앱 확장 방향을 고정한다.

## 1. 핵심 원칙

미리룩은 단순 헤어 필터가 아니라 고객, 미용사, 커뮤니티가 같은 시각 자료를 기준으로 스타일을 결정하는 플랫폼이다.

초기 웹 MVP는 남성 헤어 추천과 미용사용 상담 보드를 완성한다. 이후 고객 계정, 히스토리, 관리자, 입점, 커뮤니티, 투표, 여성 헤어, 모바일 앱으로 확장한다.

## 2. 화면 구조

### 고객 화면

- 사진 업로드
- 헤어스타일 추천
- 헤어 컬러/요청사항 선택
- 9컷 상담 보드 생성
- 결과 히스토리
- PDF/이미지 저장
- 이메일, 카카오톡, 링크 공유
- 커뮤니티 글쓰기/댓글/투표 요청
- 미용실/디자이너 검색 및 예약

### 관리자 화면

- 사용자 관리
- 생성 요청/실패 관리
- 신고/삭제 요청 관리
- 커뮤니티 모더레이션
- 미용실 입점 승인
- 디자이너 입점 승인
- 리뷰 관리
- 결제/푸시/투표 요청 관리

### 미용실/디자이너 화면

- 미용실 프로필
- 디자이너 프로필
- 시술 가능 스타일
- 위치/지도/영업시간
- 예약 가능 시간
- 고객 상담 보드 확인
- Before / AI / After 포트폴리오
- 리뷰 답변 및 관리

## 3. 고객 히스토리

고객은 본인이 받았던 추천과 생성 이미지를 히스토리로 관리할 수 있어야 한다.

필수 데이터:

- 요청 일시
- 업로드 사진
- 선택한 헤어스타일
- 선택한 헤어 컬러
- 고객 요청사항
- 추천 9장
- 상담용 9컷 보드
- 미용사 공유 여부
- 피드백
- 실제 시술 후 사진

## 4. Export / 공유

결과물은 미용실 상담에 바로 쓸 수 있어야 한다.

필수 기능:

- 9컷 상담 보드 이미지 저장
- 추천 보고서 PDF 저장
- 개별 사진 저장
- 이메일 공유
- 카카오톡 공유
- 공유 링크 생성
- 공유 링크 만료/회수

진행: PDF 저장, 개별 이미지 저장, 이메일 전송, 공유 링크 생성, 카카오톡/시스템 공유,
공유 링크 만료와 회수 동선을 연결했다. 공유 링크 회수는 고객 히스토리에서 생성한 링크를
즉시 `revoked_at` 처리해 기존 URL 열람을 차단한다.

## 5. 입점 / 예약 / 지도

미용실과 헤어 디자이너는 별도 입점 구조를 가진다.

미용실 정보:

- 상호명
- 위치/지도
- 영업시간
- 가격표
- 사진
- 예약 가능 여부
- 리뷰

디자이너 정보:

- 이름/닉네임
- 전문 스타일
- 포트폴리오
- 가능한 시술
- 예약 가능 시간
- 리뷰

예약 기능:

- 날짜/시간 선택
- 디자이너 선택
- 상담 보드 첨부
- 예약 상태 관리

진행: 입점 신청, 예약 문의, 리뷰 접수 화면과 API를 연결했다. 관리자 화면에서 입점 신청을
승인하면 고객용 살롱/디자이너 프로필이 생성되고, 리뷰를 승인하면 평점·리뷰 수와 고객 화면
리뷰 하이라이트가 갱신된다. `/salons`는 디자이너별 예약 가능 시간표, 가능 시술, 포트폴리오
예시, 상담 보드 링크 첨부, Kakao/Google 지도 키 기반 정식 임베드와 좌표 fallback을 지원한다.
남은 일은 실제 예약 확정/변경 캘린더와 파트너가 직접 시간표를 관리하는 운영 화면이다.

## 6. 커뮤니티

커뮤니티는 미용실 입점 기능과 분리한다.

방향:

- 블라인드처럼 익명 기반
- 인스타그램처럼 이미지 중심
- 댓글, 좋아요, 저장
- 신고/차단/관리자 모더레이션

초기에는 공개 커뮤니티보다 비공개 파일럿 피드백과 신뢰 기반 투표를 먼저 만든다.

## 7. 스타일 투표

사용자는 본인이 추천받은 스타일을 다른 사용자에게 투표 요청할 수 있다.

기능:

- 남성은 여성에게 투표 요청
- 여성은 남성에게 투표 요청
- 투표 대상 스타일 선택
- 투표 목적 선택: 데이트, 면접, 프로필, 일상, 소개팅
- 결제 후 푸시 알림 발송
- 투표 결과 집계

진행: 투표 요청 생성 시 요청자 성별을 기준으로 대상 성별을 자동 저장한다. 남성 요청은
여성 대상, 여성 요청은 남성 대상, 기타/비공개 요청은 전체 대상으로 처리한다. 커뮤니티
피드의 투표자는 본인 성별을 선택하고, 서버는 게시글의 대상 성별과 맞지 않는 투표를
저장하지 않는다.

메시지 정책:

- 요청자가 DM 허용을 켜면 투표자가 인스타그램 DM처럼 메시지를 보낼 수 있다.
- 요청자가 DM 비허용을 선택하면 투표와 댓글만 가능하다.
- 댓글은 구조화된 태그 중심으로 먼저 시작하고, 자유 댓글은 모더레이션 도입 후 확장한다.

## 8. 여성 헤어 확장

현재 MVP는 남성 헤어 중심이다. 여성 사진 업로드 시에도 추천 기능을 제공해야 한다.

확장 항목:

- 단발, 중단발, 장발
- 레이어드컷, 허쉬컷, 보브컷
- 웨이브펌, 볼륨매직, C컬/S컬
- 앞머리 유무
- 염색 톤
- 퍼스널 컬러 참고
- 얼굴형/분위기 설명

여성 헤어는 변수와 기대치가 높으므로 남성 MVP 파일럿 이후 별도 추천 카탈로그와 프롬프트를 만든다.

## 9. 모바일 앱 방향

앱은 블라인드와 인스타그램의 장점을 섞는다.

블라인드에서 가져올 것:

- 익명 글쓰기
- 투표
- 댓글
- 신고/차단
- 커뮤니티 탭

인스타그램에서 가져올 것:

- 이미지 중심 피드
- 스토리형 결과 공유
- DM
- 스타일 저장
- 프로필/포트폴리오

초기 모바일 앱은 Expo React Native로 만들고, 웹 MVP의 API와 DB를 재사용한다.
